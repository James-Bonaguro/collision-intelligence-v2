How It Works

The simple version of what's under the hood.

You don't need to understand how any of this works to use it. But you might want to — if only so you can explain it to your partner, your lender, or your insurance rep when they ask. Here's the whole thing in three sections, one page.

The machinery

Three layers. That's it.

Every agent works the same way. Something happens in your shop. An AI reads it and decides what to do. The decision shows up somewhere you can see.

Layer 1 · The Trigger

Something happens in your shop.

A new estimate is saved. A sublet invoice lands in email. A completion date changes. A VIN is decoded as an EV. These are real events that already happen — we just listen for them.

estimate_saved email_received ecd_changed vin_decoded
↓
Layer 2 · The Agent

An AI reads the event and decides what to do.

The agent is an instance of Claude — Anthropic's AI model, the one behind Claude Code. It receives the event, pulls relevant data (your DRP rules, the vehicle's value, the official labor-time rules), and decides on an action. Think of it as a very fast, very thorough service advisor who has read every manual.

pull DRP agreement query P-Pages calculate markup draft action
↓
Layer 3 · The Output

The decision shows up where you can see it.

The agent doesn't work in secret. Every action shows up somewhere visible: a Slack message, a line added to an estimate, a text to a customer, a note in your insurer portal. You see it. You can review it. You can undo it.

slack notification estimate line added SMS to customer portal note
Plain-English analogy
Think of the agent as a very senior service advisor who never sleeps, has read every P-Page, memorized every DRP agreement, and responds to every event in your shop within seconds — but can only do things you already explicitly authorized them to do.

Trust & Security

What the agent can and can't see.

Fair question. Here's the honest answer, no asterisks.

✓
Your customer data stays inside your existing software
The agent reads events from your estimating software the same way any integration does — through authorized APIs. Nothing is copied to our servers permanently.
✓
You can review every action taken
Every line the agent adds, every text it sends, every portal note it posts — there's an audit log. You or your manager can see exactly what happened and when.
✓
You can kill the agent in one click
Something goes wrong, something doesn't feel right — there's a pause button. The agent stops immediately. No contracts, no tickets.
✓
No training on your data
Your shop's data is not used to train any AI model. Not ours, not Anthropic's, not anyone's. Standard enterprise-grade data handling.
Common questions

What shop owners usually ask first.

Do I have to replace my estimating software?
No. The agent works with whatever you're using — CCC ONE, Mitchell, Audatex. It reads the data, it doesn't replace the tool. Your estimators keep doing their jobs the same way.
What if my staff doesn't want to use it?
They don't have to "use" anything. The agent acts in the background. Your estimator sees a notification, glances, moves on. That's the interaction. No software to learn, no buttons to click.
What happens if the agent gets something wrong?
In the first four weeks, the agent runs in "shadow mode" — every action is a suggestion your estimator approves. We use those four weeks to tune it. Once it's live, every action is logged and reversible. If it makes a mistake, we see it, we fix it, we refine the agent.
What about my DRP agreements? Can the agent really read them?
Yes. We upload your contracts during Week 1 setup and the agent queries them in real-time. If a carrier changes their DRP terms, you update the document and the agent picks it up automatically.
Is this compliant with how insurers want me to operate?
Every action the agent takes is something your staff would be doing anyway if they had infinite time. Adding P-Page lines with citations is standard practice. Extending rentals through ARMS is standard practice. The agent just does it faster and more consistently. If anything, insurer audits get cleaner because there's a perfect log.
